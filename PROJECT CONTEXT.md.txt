# PROJECT_CONTEXT.md — MVC Complaint Management System---## What This Project IsASP.NET Core MVC Citizen Complaint Management System.Citizens can submit and track complaints. Admins manage and resolve them.## Tech Stack- ASP.NET Core MVC (.NET 10)- Entity Framework Core + SQL Server Express- Bootstrap 5- Cookie Authentication (no JWT)- SignalR — real-time bell notifications for Admin- Passwords stored as plain text (no hashing — intentional for learning)---## Roles| Role | Permissions ||---|---|| Citizen | Register, Login, Submit complaints, View/Edit OWN pending complaints only || Admin | View ALL complaints, Edit status + AdminNote, Delete complaints, Real-time notifications |---## Database- Server: `localhost\\\\SQLEXPRESS`- Database: `ComplaintSystemDB`- Tables: `Users`, `Complaints`, `Notifications`---## All Migrations (Applied — DO NOT RE-RUN)1. `20260325105703\\\_InitialCreate` — Creates `Complaints` and `Users` tables2. `20260404155504\\\_AddNotifications` — Creates `Notifications` table3. `20260405145319\\\_AddAdminNoteToComplaint` — Adds `AdminNote` (nullable) column to `Complaints`---## Complete File List### Controllers| File | Purpose ||---|---|| `AccountController.cs` | Register, Login, Logout (cookie auth) || `ComplaintsController.cs` | Full CRUD + CitizenEdit + RedirectToPortal + SignalR push on new complaint || `NotificationsController.cs` | Admin-only: GetUnreadCount, GetAll, MarkAllRead |### Models| File | Fields ||---|---|| `Complaint.cs` | Id, Title, Description, Category, Status, SubmittedBy, datetime, UpdatedOn, AdminNote || `User.cs` | Id, FullName, Email, PasswordHash, Role, CreatedOn || `Notification.cs` | Id, Message, IsRead, CreatedOn |### Services- `IUserService.cs` + `UserService.cs` — EmailExists, RegisterUser, GetUserByEmailAndPassword- `IComplaintService.cs` + `ComplaintService.cs` — GetAll, GetByUser, GetById, Add, UpdateStatus, UpdateComplaintDetails, Delete- `INotificationService.cs` + `NotificationService.cs` — AddNotification, GetAllNotifications, GetUnreadCount, MarkAllAsRead, DeleteNotification### Hubs- `NotificationHub.cs` — Empty SignalR hub. Server pushes `"ReceiveNotification"` to all clients.### Data & Config- `AppDbContext.cs` — DbSets: Complaints, Users, Notifications- `Program.cs` — Full setup: SignalR, all 3 services (Scoped), cookie auth, hub mapped at `/notificationHub`- `appsettings.json` — SQL Server connection string (`ComplaintSystemDB`)### Views (Razor .cshtml)| View | Notes ||---|---|| `Login.cshtml` | Plain email + password form || `Register.cshtml` | FullName, Email, Password — always registers as Citizen || `Index.cshtml` | Admin sees all complaints; Citizen sees only their own || `Create.cshtml` | Citizen submits new complaint || `Details.cshtml` | Shows all fields incl. AdminNote + UpdatedOn; Admin sees Edit Status button || `Edit.cshtml` | Admin updates Status + AdminNote || `CitizenEdit.cshtml` | Citizen edits Title, Description, Category — only if Status == "Pending" || `RedirectToPortal.cshtml` | Post-submit page with category-based official portal link + Edit button || `\\\_Layout.cshtml` | Gov-styled layout, bell icon (Admin only), SignalR JS client loaded |### Static Files- `site.css` — Full government theme using CSS variables- `site.js` — Notification bell logic: SignalR listener, fetch unread count, dropdown, mark all read---##
