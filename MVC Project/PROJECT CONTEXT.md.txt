# MVC Complaint System — Master Context

## Project Overview
ASP.NET Core MVC Complaint Management System
- Citizens register, login, submit complaints
- Admins view all complaints, edit status, delete
- Built with .NET 10, EF Core, SQL Server, Bootstrap 5
- Cookie Authentication
- Password stored as plain text (PasswordHash field used, no hashing applied yet)

## Features Completed

### Feature 1 — Citizen sees only their own complaints
- ComplaintsController Index() checks User.IsInRole("Admin")
- Admin → GetAllComplaints()
- Citizen → GetComplaintsByUser(User.Identity.Name)
- ComplaintService.GetComplaintsByUser() filters by SubmittedBy == userName

### Feature 2 — Citizen can edit their own complaint
- New CitizenEdit GET and POST actions in ComplaintsController
- Citizens can only edit Title, Description, Category
- Status field is locked for citizens
- New CitizenEdit.cshtml view created
- Index.cshtml shows Edit button for citizens → goes to CitizenEdit
- Index.cshtml shows Edit button for Admin → goes to Admin Edit (status only)
- ComplaintService.UpdateComplaintDetails() handles citizen edit save
- UpdatedOn timestamp is set on every citizen edit

### Feature 3 + 5 — Smart portal redirect after complaint submit
- After Create POST, redirects to RedirectToPortal action
- Category-based portal URL logic in ComplaintsController:
  - Roads / Water / Sanitation → https://cmwssb.tn.gov.in/complaints-grievance
  - Electricity → https://www.tnebltd.gov.in/cgrfonline/
  - Default / Other → https://maduraicorporation.co.in/
- RedirectToPortal.cshtml shows:
  - Success message with Category and Complaint ID
  - Button to open official govt portal in new tab
  - Button to view and edit the submitted complaint (CitizenEdit)
  - Link to go back to All Complaints

### Feature 4 — Government UI Overhaul
- _Layout.cshtml updated with:
  - Top strip: "Government of India | Official Citizen Services Portal"
  - Gov header with CS emblem, portal title, ministry subtitle
  - Nav bar with dark government blue color
  - Logged-in user name shown in nav
  - Footer with copyright and portal name
- All views use card-based layout
- Consistent Bootstrap 5 badge colors for complaint status:
  - Pending → bg-warning text-dark
  - InProgress → bg-primary
  - Resolved → bg-success
  - Rejected → bg-danger

## Complaint Model Fields
- Id (int, PK, auto)
- Title (string, required, max 200)
- Description (string, required)
- Category (string, required) — Roads / Water / Electricity / Sanitation / Other
- Status (string, default "Pending") — Pending / InProgress / Resolved / Rejected
- SubmittedBy (string, required) — set from User.Identity.Name on submit
- datetime (DateTime) — set to DateTime.Now on submit
- UpdatedOn (DateTime?, nullable) — set when Admin or Citizen edits the record

## User Model Fields
- Id (int, PK, auto)
- FullName (string, required)
- Email (string, required, email format)
- PasswordHash (string, required) — plain text currently, no hashing applied
- Role (string, default "Citizen")
- CreatedOn (DateTime, default DateTime.Now)

## Portal URL Routing Logic (ComplaintsController)
  if (category == "Roads" || category == "Water" || category == "Sanitation")
      portalUrl = "https://cmwssb.tn.gov.in/complaints-grievance";
  else if (category == "Electricity")
      portalUrl = "https://www.tnebltd.gov.in/cgrfonline/";
  else
      portalUrl = "https://maduraicorporation.co.in/";

## Program.cs Setup
- AddControllersWithViews
- AddDbContext with SQL Server connection string from appsettings.json
- AddScoped IUserService → UserService
- AddScoped IComplaintService → ComplaintService
- AddAuthentication with CookieAuthenticationDefaults
  - LoginPath = /Account/Login
  - ExpireTimeSpan = 8 hours
- Middleware order: UseHttpsRedirection → UseStaticFiles → UseRouting → UseAuthentication → UseAuthorization
- Default route: controller=Complaints, action=Index
